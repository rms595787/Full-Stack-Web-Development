# Starting postgres in terminal

- psql -d postgres

## 🔑 Keywords & Meaning

### `psql`

**What it is:** PostgreSQL’s interactive terminal (command-line tool).

**What it does:** Lets you connect to a PostgreSQL database, run SQL commands, manage users, roles, and databases.  
Think of it like: the “shell” for talking to PostgreSQL.

### `-d`

**What it is:** A command-line option (flag) for specifying the database name.

**What it does:** Tells psql which database to connect to.
Example:

```bash
psql -d mydb
```

→ Connects to the database mydb.

### `postgres (after -d)`

**What it is:** The name of the database.

**What it does:** Specifies the target database to connect to.

By default, PostgreSQL creates a database named postgres during installation for administrative tasks.

In your case:

```bash
psql -d postgres

```

connects to the database named postgres.

## NOTE

### `⚡ Hidden Defaults (if you don’t provide -d or -U)`

If -d is missing → psql assumes the database name is the same as your OS username.

Example: running just psql would try to connect to database rahulsh9956.

If -U is missing → psql assumes the role/user is the same as your OS username.

Example: psql -d postgres means:

Database → postgres

Role/User → rahulsh9956
