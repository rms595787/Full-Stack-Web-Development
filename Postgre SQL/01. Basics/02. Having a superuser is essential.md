# Having a SuperUser is essential

### `psql -d postgres`

## `🧑‍💻 Your Current Situation`

You’re connecting with your macOS username role (rahulsh9956).

That role was automatically created by PostgreSQL when you installed it.

By default, this role usually `has login privileges only, not superuser powers`.

That’s why when you try certain admin tasks (like creating roles, databases, or changing privileges), you may run into “permission denied” errors.

## 🔑 Should You Create a Superuser Role?

✅ Yes, you typically should have at least one superuser role.

Here’s why:

- Superuser role is like the “root” account in Linux — it can override all permissions.
- Required for:
- - Creating and managing roles.
- - Creating and dropping databases.
- - Changing ownership and permissions.
- - Running certain extensions.
- - Without it, you’ll always be limited.

## 🛠️ How to Create a Superuser Role

1. First, connect to PostgreSQL as the user that does have rights (in your case, `rahulsh9956` works for now because it can connect to the `postgres` database).

```bash
psql -d postgres
```

2. Run this SQL to create a superuser role:

```sql
CREATE ROLE myadmin WITH LOGIN SUPERUSER PASSWORD 'mypassword';
```

- `myadmin` → your new role name (you can call it postgres too if you want the traditional name).
- `LOGIN` → makes it possible to connect.
- `SUPERUSER` → gives full powers.
- `PASSWORD 'mypassword'` → sets a password for login.

3. Exit psql:

```bash
\q
```

4. Now you can connect as your superuser:

```bash
psql -U myadmin -d postgres
```

## ⚠️ Important Notes

- You don’t need multiple superusers for daily use. Best practice is:
- - Keep one superuser role (like `postgres` or `myadmin`).
- - Use normal roles for daily work.
- - Use the superuser role only when needed.
