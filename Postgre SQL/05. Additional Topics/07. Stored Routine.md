# STORED Routine

- An SQL statement or a set of SQL Statements that can be stored on database server which can be call no. of times.
- Stored routines are precompiled blocks of SQL code that are stored in the database and can be reused whenever needed.
- They help in modular programming, reduce code repetition, and improve performance.
  In PostgreSQL, stored routines include Stored Functions and Stored Procedures.

## Types of STORED Routine

- STORED Procedure
- User defined Functions

1. `STORED Procedure`
   Set of SQL statements and procedural logic that can perform operations such as inserting, updating, deleting, and quering data.

   - Do not necessarily return a value.
   - Invoked using CALL.
   - Often used for actions like inserting/updating multiple tables, transaction handling.

     `Syntax:`

     ```sql
     CREATE PROCEDURE procedure_name(param1 type, param2 type)
     LANGUAGE plpgsql
     AS $$
     BEGIN
         -- SQL statements
     END;
     $$;
     ```

     `Function Example:`

     ```sql
     -- for updating something
     CREATE OR REPLACE PROCEDURE update_emp_salary(
         p_employee_id INT, p_new_salary INT
     )
     LANGUAGE plpgsql
     AS $$
     BEGIN
         UPDATE employees
         SET salary = p_new_salary
         WHERE emp_id = p_employee_id;
     END;
     $$;

     -- for inserting something
     CREATE OR REPLACE PROCEDURE add_employee(
        p_fname VARCHAR,
        p_lname VARCHAR,
        p_email VARCHAR,
        p_dept VARCHAR,
        p_salary INT
     )
     LANGUAGE plpgsql
     AS $$
     BEGIN
        INSERT INTO employee(fname, lname, email, dept, salary)
        VALUES (p_fname, p_lname, p_email, p_dept, p_salary);
     END;
     $$;
     ```

     `Usage:`

     ```sql
     CALL update_emp_salary(3,71000);
     ```

     `Drop:`

     ```sql
     DROP PROCEDURE update_emp_salary;
     ```

2. `USER DEFINED FUNCTIONS`

   Custom function created by the user to perform specific operations and return a value.

   - Do not necessarily return a value.
   - Invoked using CALL.
   - Often used for actions like inserting/updating multiple tables, transaction handling.

   `Syntax:`

   ```sql
   CREATE FUNCTION function_name(param1 type, param2 type)
   RETURNS return_type AS $$
   BEGIN
       -- SQL statements
             RETURN some_value;
   END;
   $$ LANGUAGE plpgsql;
   ```

   `Function Example:`

   Use Case - Find name of the employees in each department having maximum salary.

   We want to achieve this code using functions, because writing this query multiple times is not an easy task

   ```sql
      SELECT e.emp_id, e.fname, e.salary
      FROM employee e
      WHERE e.dept='HR' AND e.salary = (
          SELECT MAX(emp.salary)
          FROM employee emp
          WHERE emp.dept = e.dept
      );
   ```

   `Required Function:`

   ```sql
   CREATE OR REPLACE FUNCTION dept_max_sal_emp(dept_name VARCHAR)
   RETURNS TABLE(emp_id INT, fname VARCHAR, salary INT)
   AS $$
   BEGIN
      RETURN QUERY
      SELECT e.emp_id, e.fname, e.salary
      FROM employee e
      WHERE e.dept = dept_name AND e.salary = (
          SELECT MAX(emp.salary)
          FROM employee emp
          WHERE emp.dept = dept_name
      );
   END;
   $$ LANGUAGE plpgsql;
   ```

   `USAGE OF FUNCTION:`

   ```sql
   SELECT * FROM dept_max_salary_emp('HR');
   ```

   `DROP:`

   ```sql
   DROP FUNCTION dept_max_salary_emp;
   ```

### 🔹 Advantages

- Reusability → Write once, call many times.
- Performance → Precompiled and cached in the database.
- Maintainability → Business logic stored centrally in the DB.
- Security → Permissions can restrict direct table access but allow routine execution.

# Note

## 🔹 Key Points about Delimiters in PostgreSQL

1. \$$ **is just a convention**, not mandatory.  
   You can **use any delimiter** like `$func$, $body$, $abc$,` etc.  
   Example:
   ```sql
   CREATE FUNCTION add_numbers(a INT, b INT)
   RETURNS INT AS $func$
   BEGIN
       RETURN a + b;
   END;
   $func$ LANGUAGE plpgsql;
   ```
2. Why delimiters are needed?
   - To clearly separate the function body (which may contain `'`, `"` and other special characters) from the SQL command itself.
   - This avoids conflicts with string literals inside the function.
3. Which delimiters you CANNOT use?
   You **cannot use a plain single quote** `'` or **double quote** `"` as a delimiter → because they already have meaning in SQL (string literals and identifiers).
   You also cannot use mismatched delimiters (like $abc to start and $xyz$ to end).
4. ✅ Valid examples of delimiters:
   - `$$ ... $$`
   - `$myfunc$ ... $myfunc$`
   - `$code$ ... $code$`
5. 🚫 Invalid delimiters:
   - `' ... '` (string literal, not a delimiter)
   - `" ... "` (used for identifiers like column/table names)
   - `$abc$ ... $def$` (must match)

## 🔹 Why people usually use $$?

- It’s short, clear, and doesn’t conflict with most function bodies.
- But if your function contains `$$` inside (rare case), you can choose another symbol like `$proc$`.
