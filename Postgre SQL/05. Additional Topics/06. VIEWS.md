# 🔹 VIEWS

- A View in PostgreSQL is a virtual table that is based on the result of a SQL query.
- It does not store the actual data itself; instead, it stores the query definition and retrieves data dynamically whenever it is used.
- Views are used to simplify complex queries, enhance security, and provide a layer of abstraction.

## 🔹 Syntax: Create a View

```sql
CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;
```

## 🔹 Example

Suppose we have a students table:

```sql
id | name   | age | grade
---+--------+-----+-------
 1 | Rahul  | 20  | A
 2 | Priya  | 19  | B
 3 | Aman   | 21  | A
```

`Create a View for Students with grade A`

```sql
CREATE VIEW top_students AS
SELECT name, age
FROM students
WHERE grade = 'A';
```

`Create a View for students with grade 'A':`

```sql
SELECT * FROM top_students;
```

`Output`

```sql
name   | age
-------+-----
Rahul  | 20
Aman   | 21
```

## 🔹 Benefits of Views

- ✅ Simplifies queries → Encapsulates complex SQL.
- ✅ Security → Can restrict access to sensitive columns.
- ✅ Data abstraction → Hides table structure details.
- ✅ Reusability → Can be reused in multiple queries.

## 🔹 Types of Views

- Simple View – Based on a single table without functions or groups.
- Complex View – Based on multiple tables, with joins, groups, and aggregate functions.
- Materialized View – Stores the actual query result physically and can be refreshed. Useful for performance.

## Drop a View

```sql
DROP VIEW view_name;
```
