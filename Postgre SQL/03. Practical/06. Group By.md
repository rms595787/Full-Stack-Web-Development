# GROUP BY in PostgreSQL

## 📌 Definition

The **GROUP BY** clause in PostgreSQL is used to arrange identical data into groups.  
It is often used with **aggregate functions** (like COUNT, SUM, AVG, MIN, MAX) to perform calculations for each group separately.

---

## ✅ Syntax

```sql
SELECT column_name, aggregate_function(column_name)
FROM table_name
GROUP BY column_name;
```

#### 1. Count employees in each department

```sql
SELECT dept, COUNT(*)
FROM employee
GROUP BY dept;
```

📌 Result: Shows how many employees are in each department.  
![Result](./_images/Group%20By1.png)

#### 2. Average salary per department

```sql
SELECT dept, AVG(salary)
FROM employee
GROUP BY dept;
```

📌 Result: Returns average salary grouped by department.
![Result](./_images/Group%20by2.png)

#### 3. Minimum and Maximum salary in each department

```sql
SELECT dept, MIN(salary) AS min_salary, MAX(salary) AS max_salary
FROM employee
GROUP BY dept;
```

📌 Result: For each department, shows the lowest and highest salary.
![Result](./_images/Group%20by3.png)

#### 4. Using GROUP BY with multiple columns

```sql
SELECT dept, hire_date, COUNT(*)
FROM employee
GROUP BY dept, hire_date;
```

📌 Result: Groups employees by both department and hire date.
![Result](./_images/Group%20by4.png)
Here we got all unique pairs of dept and hire_date

#### ⚡ Key Points

GROUP BY is always written after WHERE and before ORDER BY.
Every column in the SELECT clause must either:
Appear in the GROUP BY clause, OR
Be used inside an aggregate function.
Useful for generating summarized reports.
