# Establishing connectivity b/w terminal and pgadmin

---

## 1️⃣ PostgreSQL Client-Server Architecture

- PostgreSQL uses a **client-server model**:
  - **Server**: Stores databases and handles queries.
  - **Client**: Any tool or program that connects to the server (`psql`, `pgAdmin4`, Python scripts, etc.)
- Multiple clients can connect simultaneously to the same server.

---

## 2️⃣ Connecting to PostgreSQL

### Terminal (`psql`)

```bash
psql -h <host> -p <port> -U <username> -d <database>
```

- -h: Host address (localhost or IP for remote)
- -p: Port (default 5432)
- -U: Username/role
- -d: Database name (optional, defaults to username)

**Example:**

```bash
psql -h localhost -p 5432 -U postgres -d mydb
```

### pgAdmin4

- Register a server:
  1. Right-click Servers → Create → Server
  2. Enter Host name/address, Port, Username, and Password
- Once connected, you can browse databases, roles, and tables.
  Note: pgAdmin4 is a GUI client — it does not store databases itself.

---

## 3️⃣ Users, Roles, and Privileges

- Roles = Users + Groups (can own objects and have permissions)
- Check current role:

```sql
SELECT current_user;   -- Current active role
SELECT session_user;   -- Original login role
```

- List all roles:

```sql
\du   -- in psql shell
-- OR
SELECT * FROM pg_roles;
```

- Assign roles:

```sql
GRANT role_name TO user_name;
```

---

## 4️⃣ Key PostgreSQL System Parameters

| Parameter  | Description                               |
| ---------- | ----------------------------------------- |
| `host`     | Server address (localhost, IP)            |
| `port`     | Port PostgreSQL listens on (default 5432) |
| `dbname`   | Database name to connect                  |
| `user`     | Role for authentication                   |
| `password` | Role password (if required)               |

Tip: Multiple PostgreSQL clusters can run on different ports — ensure you connect to the correct one.

## 5️⃣ Checking Server & Connection Info

```sql
SELECT inet_server_addr(), inet_server_port(), current_user;
```

- inet_server_addr() → server IP
- inet_server_port() → server port
- current_user → role currently in use

## 6️⃣ Other Important Connectivity Concepts

1. Localhost vs Remote Host

   - localhost → same machine
   - IP/hostname → remote machine
   - Make sure PostgreSQL allows remote connections (check pg_hba.conf).

2. Ports
   - Default: 5432
   - Different clusters may use other ports (e.g., 5433).
3. Session vs Login Role
   - session_user → original login
   - current_user → active role (can change with SET ROLE)
4. Authentication Methods
   - md5 → password required
   - trust → no password
   - Configured in pg_hba.conf
