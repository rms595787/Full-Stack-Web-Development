## Datatypes

An attribute that specifies the type of data in a column of our database - table.

`For more details check out this file - `
[Data Types](../02.%20Theory/03.%20Data%20Types.md)

### Most widely used are

- Numeric - INT, DOUBLE, FLOAT, DECIMAL
- String - VARCHAR
- Date - DATE
- Boolean - BOOLEAN

Decimal

- Syntax for DECIMAL -> DECIMAL(5,2)
- 5 Total number of digits (including digits which are after decimal)
- 2 Total number of digits after decimal

## Constraints

A contraint in PostgreSql is a rule applied to a column.

`For more details on constrainst checkout this file - `
[Constraints](../02.%20Theory/06.%20Constraints.md)

`For more details on working with constraints checkout this file - `
[Alter Table and Constraints](../02.%20Theory/07.%20Alter%20Table.md)

1.  `Primary Key`

    - The PRIMARY KEY constraint uniquely identifies each record in a table.
    - Primary keys `must contain unique values`, and `cannot` contain `NULL` values.
    - A table can have only `one` primary key.

2.  `NOT NULL`

    Ex -

    CREATE TABLE customers(  
    id INT NOT NULL;  
    name VARCHAR(100) NOT NULL;  
    );

3.  `Default Value`  
     Ex -  
     CREATE TABLE customers(  
     acc_no INT PRIMARY KEY,  
     name VARCHAR(100) NOT NULL,  
     acc_type VARCHAR(50) NOT NULL DEFAULT 'Savings'  
     );

4.  For AUTO_INCREMENT in postgres we use keyword `SERIAL`

    Ex -  
    CREATE TABLE employee (  
    id SERIAL PRIMARY KEY,  
    firstname VARCHAR(50),  
    lastname VARCHAR(50)  
    )
