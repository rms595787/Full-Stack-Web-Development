# Check

A **CHECK constraint** ensures that data entered into a column meets a specific condition before it gets stored in the table. If the condition evaluates to FALSE, PostgreSQL rejects the insert or update.

## Check Syntax

```sql
CREATE TABLE table_name (
  column_name data_type CHECK (condition)
);
```

## Example 1

```sql
CREATE TABLE contacts(
  name VARCHAR(50),
  mob VARCHAR(15) UNIQUE CHECK (LENGTH(mob)>=10)
);
```

## Example 2: Single Column Check

```sql
CREATE TABLE employee (
    emp_id SERIAL PRIMARY KEY,
    name VARCHAR(50),
    age INT CHECK (age >= 18)   -- ensures employee must be 18 or older
);
```

## Example 3: Add a check later

```sql
ALTER TABLE employee
ADD CONSTRAINT salary_check CHECK (salary > 0);
```

## Example 4: Multiple Columns Check

```sql
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    quantity INT,
    price NUMERIC,
    CHECK (quantity > 0 AND price > 0)  -- ensures valid positive values
);
```

## Example 5: Drop constraint

```sql
ALTER TABLE contacts
DROP CONSTRAINT mob_no_less_than_10digits;
```

This "mob_no_less_than_10digits" is nothing but name assigned by postgres to the check condition you easily know by displaying table. properties.

## Example 6: Named Constraints

```sql
ALTER TABLE person
ADD CONSTRAINT person_mob_check CHECK(LENGTH(mob)>=10);
```
