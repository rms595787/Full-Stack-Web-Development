# PostgreSQL Roles & Permissions – Cheat Sheet

## 1. Role Basics
- A **role** = user or group of users.
- Roles can **own objects**, **log in**, and have **privileges**.

### Create Role
```sql
CREATE ROLE role_name;
```

### Create User (role with LOGIN)
```sql
CREATE USER username WITH PASSWORD 'secret';
```

---

## 2. Role Attributes
- `LOGIN` – can log in
- `SUPERUSER` – full power (bypass all checks)
- `CREATEDB` – can create databases
- `CREATEROLE` – can create/alter roles
- `INHERIT` – inherits privileges from roles it’s a member of
- `REPLICATION` – can initiate replication
- `BYPASSRLS` – bypass row-level security

Example:
```sql
CREATE ROLE dev LOGIN PASSWORD 'pass' CREATEDB;
```

---

## 3. Managing Roles
- Change role:
```sql
ALTER ROLE role_name [options];
```

- Delete role:
```sql
DROP ROLE role_name;
```

- Change password:
```sql
ALTER ROLE role_name PASSWORD 'newpass';
```

---

## 4. Granting Privileges
### Database
```sql
GRANT ALL PRIVILEGES ON DATABASE dbname TO role_name;
```

### Table
```sql
GRANT SELECT, INSERT, UPDATE, DELETE ON table_name TO role_name;
```

### Schema
```sql
GRANT USAGE, CREATE ON SCHEMA schema_name TO role_name;
```

### Revoke
```sql
REVOKE privilege FROM role_name;
```

---

## 5. Role Membership
- Add role to group:
```sql
GRANT group_role TO user_role;
```

- Remove:
```sql
REVOKE group_role FROM user_role;
```

- Check membership:
```sql
\du
```

---

## 6. Useful Commands
- List roles:
```sql
\du
```

- Current role:
```sql
SELECT current_user;
```

- Switch role:
```sql
SET ROLE role_name;
```

- Reset role:
```sql
RESET ROLE;
```

---

✅ Use **groups of roles** for clean privilege management.  
✅ Keep `SUPERUSER` usage minimal.  
✅ Always follow **principle of least privilege**.
