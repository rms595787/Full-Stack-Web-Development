# Operators in Postgres

## 1. Arithmetic Operators

| S.No. | Operator | Usage               | Example                 |
| ----- | -------- | ------------------- | ----------------------- |
| 1     | `+`      | Addition            | `SELECT 10 + 5;` → `15` |
| 2     | `-`      | Subtraction         | `SELECT 10 - 5;` → `5`  |
| 3     | `*`      | Multiplication      | `SELECT 10 * 5;` → `50` |
| 4     | `/`      | Division            | `SELECT 10 / 2;` → `5`  |
| 5     | `%`      | Modulus (remainder) | `SELECT 10 % 3;` → `1`  |

---

## 2. Comparison Operators

| S.No. | Operator    | Usage                 | Example                   |
| ----- | ----------- | --------------------- | ------------------------- |
| 1     | `=`         | Equal to              | `SELECT 5 = 5;` → `TRUE`  |
| 2     | `!=` / `<>` | Not equal to          | `SELECT 5 <> 3;` → `TRUE` |
| 3     | `>`         | Greater than          | `SELECT 7 > 3;` → `TRUE`  |
| 4     | `<`         | Less than             | `SELECT 3 < 7;` → `TRUE`  |
| 5     | `>=`        | Greater than or equal | `SELECT 5 >= 5;` → `TRUE` |
| 6     | `<=`        | Less than or equal    | `SELECT 4 <= 5;` → `TRUE` |

---

## 3. Logical Operators

| S.No. | Operator | Usage                                          | Example                            |
| ----- | -------- | ---------------------------------------------- | ---------------------------------- |
| 1     | `AND`    | Returns TRUE if both conditions are true       | `SELECT TRUE AND FALSE;` → `FALSE` |
| 2     | `OR`     | Returns TRUE if at least one condition is true | `SELECT TRUE OR FALSE;` → `TRUE`   |
| 3     | `NOT`    | Negates a condition                            | `SELECT NOT TRUE;` → `FALSE`       |

---

## 4. String Operators

| S.No. | Operator | Usage                                   | Example                                         |     |     |     |     |
| ----- | -------- | --------------------------------------- | ----------------------------------------------- | --- | --- | --- | --- |
| 1     | `\|\|` | Concatenation                           | `SELECT 'Hello' \|\| ' World';`→`Hello World` |
| 2     | `LIKE`   | Pattern match (case-sensitive)          | `SELECT 'Rahul' LIKE 'Ra%';` → `TRUE`           |     |     |     |     |
| 3     | `ILIKE`  | Pattern match (case-insensitive)        | `SELECT 'rahul' ILIKE 'Ra%';` → `TRUE`          |     |     |     |     |
| 4     | `~`      | Regex match (case-sensitive)            | `SELECT 'abc' ~ 'a.c';` → `TRUE`                |     |     |     |     |
| 5     | `~*`     | Regex match (case-insensitive)          | `SELECT 'ABC' ~* 'a.c';` → `TRUE`               |     |     |     |     |
| 6     | `!~`     | Regex does not match (case-sensitive)   | `SELECT 'abc' !~ 'd';` → `TRUE`                 |     |     |     |     |
| 7     | `!~*`    | Regex does not match (case-insensitive) | `SELECT 'ABC' !~* 'd';` → `TRUE`                |     |     |     |     |

---

## 5. Bitwise Operators

| S. No. | Operator | Usage               | Example                 |            |          |
| ------ | -------- | ------------------- | ----------------------- | ---------- | -------- |
| 1      | `&`      | Bitwise AND         | `SELECT 5 & 3;` → `1`   |            |          |
| 2      | \`       | \`                  | Bitwise OR              | \`SELECT 5 | 3;`→`7\` |
| 3      | `#`      | Bitwise XOR         | `SELECT 5 # 3;` → `6`   |            |          |
| 4      | `~`      | Bitwise NOT         | `SELECT ~5;` → `-6`     |            |          |
| 5      | `<<`     | Bitwise shift left  | `SELECT 5 << 1;` → `10` |            |          |
| 6      | `>>`     | Bitwise shift right | `SELECT 5 >> 1;` → `2`  |            |          |

---

## 6. Set Operators

| S. No. | Operator    | Usage                                       | Example                                                  |
| ------ | ----------- | ------------------------------------------- | -------------------------------------------------------- |
| 1      | `UNION`     | Combines results, removes duplicates        | `SELECT dept FROM emp1 UNION SELECT dept FROM emp2;`     |
| 2      | `UNION ALL` | Combines results, keeps duplicates          | `SELECT dept FROM emp1 UNION ALL SELECT dept FROM emp2;` |
| 3      | `INTERSECT` | Returns common rows                         | `SELECT dept FROM emp1 INTERSECT SELECT dept FROM emp2;` |
| 4      | `EXCEPT`    | Returns rows from first query not in second | `SELECT dept FROM emp1 EXCEPT SELECT dept FROM emp2;`    |

---

## 7. Membership & Range Operators

| S. No. | Operator              | Usage                                 | Example                                     |
| ------ | --------------------- | ------------------------------------- | ------------------------------------------- |
| 1      | `IN`                  | Value exists in a list                | `SELECT 10 IN (5,10,15);` → `TRUE`          |
| 2      | `NOT IN`              | Value does not exist in list          | `SELECT 20 NOT IN (5,10,15);` → `TRUE`      |
| 3      | `BETWEEN ... AND ...` | Value lies within a range (inclusive) | `SELECT 15 BETWEEN 10 AND 20;` → `TRUE`     |
| 4      | `NOT BETWEEN`         | Value outside the range               | `SELECT 25 NOT BETWEEN 10 AND 20;` → `TRUE` |

---

## 8. NULL Operators

| S. No. | Operator      | Usage                       | Example                          |
| ------ | ------------- | --------------------------- | -------------------------------- |
| 1      | `IS NULL`     | Checks if value is NULL     | `SELECT NULL IS NULL;` → `TRUE`  |
| 2      | `IS NOT NULL` | Checks if value is not NULL | `SELECT 5 IS NOT NULL;` → `TRUE` |

---

## 9. Array Operators

| S. No. | Operator | Usage                     | Example                                             |
| ------ | -------- | ------------------------- | --------------------------------------------------- |
| 1      | `@>`     | Contains                  | `SELECT '{1,2,3}'::int[] @> '{2}';` → `TRUE`        |
| 2      | `<@`     | Contained in              | `SELECT '{2}'::int[] <@ '{1,2,3}'::int[];` → `TRUE` |
| 3      | `&&`     | Overlap                   | `SELECT '{1,2}'::int[] && '{2,3}'::int[];` → `TRUE` |
| 4      | `ANY`    | Compare with any element  | `SELECT 3 = ANY('{1,2,3}'::int[]);` → `TRUE`        |
| 5      | `ALL`    | Compare with all elements | `SELECT 3 > ALL('{1,2}'::int[]);` → `TRUE`          |

---

## 10. JSON Operators

| S. No. | Operator | Usage                           | Example                                                |
| ------ | -------- | ------------------------------- | ------------------------------------------------------ |
| 1      | `->`     | Get JSON object field           | `SELECT '{"name":"Rahul"}'::json->'name';` → `"Rahul"` |
| 2      | `->>`    | Get JSON field as text          | `SELECT '{"name":"Rahul"}'::json->>'name';` → `Rahul`  |
| 3      | `#>`     | Get JSON object at path         | `SELECT '{"a": {"b": "c"}}'::json#>'{a,b}';` → `"c"`   |
| 4      | `#>>`    | Get JSON object at path as text | `SELECT '{"a": {"b": "c"}}'::json#>>'{a,b}';` → `c`    |

---
