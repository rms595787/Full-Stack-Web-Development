# 🔹 What is a Clause?

In SQL, a clause is a part of a query that performs a specific task.  
A clause defines the conditions, filtering, grouping, or ordering of data.  
Clauses are always used inside a statement (like SELECT, INSERT, UPDATE, DELETE).  
👉 Think of clauses as building blocks of SQL statements.

---

## Clauses in postgres

| S.No. | Clause         | Usage / Meaning                                                                                                                 | Example                                                                              |
| ----- | -------------- | ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| 1     | **WHERE**      | Filters rows based on a condition.                                                                                              | `SELECT * FROM employees WHERE dept = 'HR';`                                         |
| 2     | **ORDER BY**   | Sorts rows in ascending (`ASC`) or descending (`DESC`) order. Supports `NULLS FIRST/LAST`.                                      | `SELECT * FROM employees ORDER BY salary DESC NULLS LAST;`                           |
| 3     | **GROUP BY**   | Groups rows that have the same values in specified columns.                                                                     | `SELECT dept, COUNT(*) FROM employees GROUP BY dept;`                                |
| 4     | **HAVING**     | Filters groups created by `GROUP BY`.                                                                                           | `SELECT dept, COUNT(*) FROM employees GROUP BY dept HAVING COUNT(*) > 5;`            |
| 5     | **LIMIT**      | Restricts number of rows returned.                                                                                              | `SELECT * FROM employees LIMIT 10;`                                                  |
| 6     | **OFFSET**     | Skips given number of rows before starting to return rows. Useful for pagination.                                               | `SELECT * FROM employees LIMIT 10 OFFSET 5;`                                         |
| 7     | **JOIN**       | Combines rows from two or more tables based on a related column. Supports `INNER`, `LEFT`, `RIGHT`, `FULL`, `CROSS`, `NATURAL`. | `SELECT e.fname, d.dept_name FROM employee e JOIN department d ON e.dept_id = d.id;` |
| 8     | **DISTINCT**   | Removes duplicate values from result set.                                                                                       | `SELECT DISTINCT dept FROM employees;`                                               |
| 9     | **FROM**       | Specifies the table(s) to retrieve data from.                                                                                   | `SELECT * FROM employees;`                                                           |
| 10    | **IN**         | Checks if a value matches any value in a list or subquery.                                                                      | `SELECT * FROM employees WHERE dept IN ('HR', 'Finance');`                           |
| 11    | **BETWEEN**    | Checks if a value lies within a range (inclusive).                                                                              | `SELECT * FROM employees WHERE salary BETWEEN 30000 AND 60000;`                      |
| 12    | **LIKE**       | Pattern matching with wildcards (`%`, `_`). Case-sensitive.                                                                     | `SELECT * FROM employees WHERE fname LIKE 'R%';`                                     |
| 13    | **ILIKE**      | Case-insensitive pattern matching (Postgres-only).                                                                              | `SELECT * FROM employees WHERE fname ILIKE 'r%';`                                    |
| 14    | **IS NULL**    | Checks for NULL values.                                                                                                         | `SELECT * FROM employees WHERE email IS NULL;`                                       |
| 15    | **FETCH**      | Alternative to `LIMIT`, SQL-standard way of restricting rows.                                                                   | `SELECT * FROM employees FETCH FIRST 5 ROWS ONLY;`                                   |
| 16    | **RETURNING**  | Postgres-only: returns values of rows affected by `INSERT`, `UPDATE`, `DELETE`.                                                 | `INSERT INTO employees(fname) VALUES('John') RETURNING emp_id;`                      |
| 17    | **WITH (CTE)** | Common Table Expressions, supports recursive queries.                                                                           | `WITH temp AS (SELECT * FROM employees WHERE dept='HR') SELECT * FROM temp;`         |
