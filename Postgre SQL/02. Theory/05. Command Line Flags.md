# PostgreSQL Command-Line Options (`-` Flags) Cheat Sheet

In PostgreSQL, many **command-line utilities** like `psql`, `pg_dump`, `pg_restore`, `createdb`, and `dropdb` use **flags starting with `-`** to control behavior, pass connection info, or set options.

These flags are **used in the terminal**, not inside SQL queries.

---

## 1️⃣ Common `psql` Command-Line Options

| Flag            | Description                            | Example                                      |
| --------------- | -------------------------------------- | -------------------------------------------- |
| `-h <host>`     | Hostname or IP address of the server   | `psql -h localhost -U postgres -d mydb`      |
| `-p <port>`     | Port number the server is listening on | `psql -p 5432 -U postgres`                   |
| `-U <username>` | Username to connect as                 | `psql -U postgres -d mydb`                   |
| `-d <dbname>`   | Database name to connect to            | `psql -d mydb -U postgres`                   |
| `-f <file>`     | Execute SQL commands from a file       | `psql -d mydb -f script.sql`                 |
| `-c "<SQL>"`    | Execute a single SQL command and exit  | `psql -d mydb -c "SELECT * FROM employees;"` |
| `-W`            | Prompt for password                    | `psql -h localhost -U postgres -W`           |
| `-a`            | Echo all input from script files       | `psql -f script.sql -a`                      |
| `-q`            | Run quietly (suppress extra output)    | `psql -d mydb -q`                            |
| `--version`     | Show the version of `psql`             | `psql --version`                             |

## 2️⃣ Common Flags for Other Utilities

### `createdb` / `dropdb`

| Flag            | Description        | Example                      |
| --------------- | ------------------ | ---------------------------- |
| `-h <host>`     | Server host        | `createdb -h localhost mydb` |
| `-p <port>`     | Server port        | `dropdb -p 5432 mydb`        |
| `-U <username>` | User to connect as | `createdb -U postgres mydb`  |

### `pg_dump`

| Flag            | Description                                                          | Example                                              |
| --------------- | -------------------------------------------------------------------- | ---------------------------------------------------- |
| `-h <host>`     | Host to connect to                                                   | `pg_dump -h localhost -U postgres mydb > backup.sql` |
| `-p <port>`     | Port number                                                          | `pg_dump -p 5432 mydb > backup.sql`                  |
| `-U <username>` | Username                                                             | `pg_dump -U postgres mydb > backup.sql`              |
| `-F <format>`   | Output format: `p` (plain), `c` (custom), `t` (tar), `d` (directory) | `pg_dump -F c -U postgres mydb > backup.dump`        |
| `-f <file>`     | Output file                                                          | `pg_dump -f backup.sql mydb`                         |
| `-a`            | Dump only data                                                       | `pg_dump -a mydb > data.sql`                         |
| `-s`            | Dump only schema                                                     | `pg_dump -s mydb > schema.sql`                       |

### `pg_restore`

| Flag            | Description                           | Example                                       |
| --------------- | ------------------------------------- | --------------------------------------------- |
| `-d <dbname>`   | Target database to restore            | `pg_restore -d mydb backup.dump`              |
| `-U <username>` | Username                              | `pg_restore -U postgres -d mydb backup.dump`  |
| `-h <host>`     | Server host                           | `pg_restore -h localhost -d mydb backup.dump` |
| `-p <port>`     | Server port                           | `pg_restore -p 5432 -d mydb backup.dump`      |
| `-c`            | Clean (drop objects before restoring) | `pg_restore -c -d mydb backup.dump`           |
| `-v`            | Verbose mode                          | `pg_restore -v -d mydb backup.dump`           |

---

## 3️⃣ Notes

1. Flags starting with `-` are **used in the terminal**, not inside SQL commands.
2. Most **connection-related flags** (`-h`, `-p`, `-U`, `-d`) are common across `psql`, `pg_dump`, `pg_restore`, `createdb`, and `dropdb`.
3. Use `--help` with any utility to see all available flags, e.g.:

```bash
psql --help
pg_dump --help
```

## 4️⃣ Combine flags as needed, e.g., psql -h localhost -p 5432 -U postgres -d mydb -f script.sql.
