# PostgreSQL Data Types Cheat Sheet

This README provides an overview of the commonly used **PostgreSQL data types** along with their default sizes and usage.

---

## 1️⃣ Numeric Types

| Data Type             | Description                                | Size / Range                                                   |
| --------------------- | ------------------------------------------ | -------------------------------------------------------------- |
| `smallint`            | 2-byte integer                             | -32,768 to 32,767                                              |
| `integer` / `int`     | 4-byte integer                             | -2,147,483,648 to 2,147,483,647                                |
| `bigint`              | 8-byte integer                             | -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807        |
| `decimal` / `numeric` | Exact numeric type, user-defined precision | Up to 131072 digits before decimal, 16383 digits after decimal |
| `real`                | 4-byte floating point                      | ~6 decimal digits precision                                    |
| `double precision`    | 8-byte floating point                      | ~15 decimal digits precision                                   |
| `serial`              | Auto-increment integer                     | 1 to 2,147,483,647                                             |
| `bigserial`           | Auto-increment bigint                      | 1 to 9,223,372,036,854,775,807                                 |

---

## 2️⃣ Character / String Types

| Data Type    | Description            | Default Size / Limit             |
| ------------ | ---------------------- | -------------------------------- |
| `char(n)`    | Fixed-length string    | n characters (padded if shorter) |
| `varchar(n)` | Variable-length string | Up to n characters               |
| `text`       | Variable-length string | Practically unlimited (~1 GB)    |

---

## 3️⃣ Boolean Type

| Data Type | Description | Values                  |
| --------- | ----------- | ----------------------- |
| `boolean` | True/False  | `TRUE`, `FALSE`, `NULL` |

---

## 4️⃣ Date / Time Types

| Data Type                       | Description        | Format / Size                                |
| ------------------------------- | ------------------ | -------------------------------------------- |
| `date`                          | Calendar date      | YYYY-MM-DD                                   |
| `time [without time zone]`      | Time of day        | HH:MM:SS                                     |
| `time with time zone`           | Time with timezone | HH:MM:SS+TZ                                  |
| `timestamp [without time zone]` | Date + Time        | YYYY-MM-DD HH:MM:SS                          |
| `timestamp with time zone`      | Date + Time + TZ   | YYYY-MM-DD HH:MM:SS+TZ                       |
| `interval`                      | Time interval      | years, months, days, hours, minutes, seconds |

---

## 5️⃣ Binary Data Types

| Data Type | Description        | Size / Limit |
| --------- | ------------------ | ------------ |
| `bytea`   | Binary / raw bytes | Up to 1 GB   |

---

## 6️⃣ UUID and JSON Types

| Data Type | Description                          | Size / Limit    |
| --------- | ------------------------------------ | --------------- |
| `uuid`    | Universally unique identifier        | 16 bytes        |
| `json`    | JSON data (text)                     | Variable length |
| `jsonb`   | Binary JSON (optimized for querying) | Variable length |

---

## 7️⃣ Array Types

- PostgreSQL supports arrays of any data type:

```sql
scores integer[] = '{10, 20, 30}';
tags text[] = '{"postgres","sql","database"}';
```

**Note:** The size depends on the number of elements in the array.

---

## 8️⃣ Other Special Types

| Data Type              | Description                   | Example                                   |
| ---------------------- | ----------------------------- | ----------------------------------------- |
| `money`                | Currency value                | `salary money = '$1000.00';`              |
| `inet` / `cidr`        | IP addresses / network blocks | `ip inet = '192.168.1.1';`                |
| `macaddr`              | MAC addresses                 | `mac macaddr = '08:00:2b:01:02:03';`      |
| `tsvector` / `tsquery` | Full-text search types        | `document tsvector = 'PostgreSQL & SQL';` |

## ✅ Notes

1. Arrays can store multiple values of any valid PostgreSQL data type.
2. Special types like money, inet, and macaddr are useful for financial, networking, and system-related data.
3. tsvector and tsquery are used for full-text search capabilities in PostgreSQL.
