# Database Notes

## What is a Database?

An **organized collection of data** that allows storing, retrieving, and manipulating information efficiently.

---

## Database vs DBMS

- **Database** → The place where data is stored.
- **DBMS (Database Management System)** → A software tool that acts as a **mediator** between the user and the database. It helps in storing, accessing, and managing data.

---

## What is RDBMS?

A **Relational Database Management System (RDBMS)** is a type of DBMS that stores data in **structured tables** (rows and columns) and uses **SQL** for querying and managing data.

### Popular Databases:

- PostgreSQL
- MySQL
- Oracle
- SQLite
- MongoDB (non-relational)
- Firebird
- Redis

---

## SQL vs PostgreSQL

- **SQL (Structured Query Language)**  
  A **language** used to communicate with databases.  
  Example:
  ```sql
  SELECT * FROM person_db;
  ```
- **PostgreSQL**
  An **open-source RDBMS** that uses SQL (and extensions like PL/pgSQL). It provides advanced features such as ACID compliance, JSON support, and extensibility.

## Connecting to PostgreSQL via Terminal

You can connect to a PostgreSQL database using the `psql` command in two ways:

1. **Without specifying role** (Uses your current OS user by default):

```bash
psql -d databaseName
```

2. **With a specific role** (User):

```bash
psql -U roleName -d databaseName
```

- `-U` -> Specifies the **role/user** to connect as (e.g., `postgres`, `admin`, `rahulsh9956`).
- `-d` -> Specifies the **database name** to connect to (e.g., `postgres`, `mydb`).

## Roles & Permissions in PostgreSQL

---

### What is a Role?

- A **role** in PostgreSQL is like a user or a group.
- Roles can **own objects** (tables, databases, etx.) and have **privileges** to access or modify them.
- Roles can also **inherit** permissions from other roles.

---

### Types of Roles

- **Superuser** → Full control, bypasses permission checks.
- **Login** Role → Can log in (`CREATE ROLE name LOGIN`).
- **Group** Role → Cannot log in, used for grouping permissions.
- **Replication** Role → Used for database replication.
- **Barman / Monitoring Roles** → Custom roles for backups or monitoring.

---

### Role Attributes

When creating a role, you can assign attributes such as:

- **LOGIN** → Can log in.
- **SUPERUSER** → Bypasses all permission checks.
- **CREATEDB** → Can create new databases.
- **CREATEROLE** → Can create/manage roles.
- **REPLICATION** → Allows replication.
- **INHERIT** → Inherits privileges from roles it’s a member of.
- **BYPASSRLS** → Bypasses Row-Level Security policies.

---

### Creating Roles

```sql
-- Create a simple login role
CREATE ROLE user1 LOGIN PASSWORD 'secret';

-- Create a role with superuser privileges
CREATE ROLE admin SUPERUSER LOGIN PASSWORD 'admin123';

-- Create a group role
CREATE ROLE dev_team NOLOGIN;

-- Add a user to a group
GRANT dev_team TO user1;
```

---

### Granting and Revoking Privileges

```sql
-- Grant privileges
GRANT SELECT, INSERT ON employees TO user1;

-- Revoke privileges
REVOKE INSERT ON employees FROM user1;
```

---

### Dropping Roles

```sql
DROP ROLE user1;
```
