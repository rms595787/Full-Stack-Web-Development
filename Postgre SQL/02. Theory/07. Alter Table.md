# PostgreSQL: ALTER TABLE Cheat Sheet

The `ALTER TABLE` command is used to **update the structure of an existing table**.  
You can add, drop, rename, or modify columns, and add or drop constraints.

---

## 1️⃣ Add a Column

```sql
ALTER TABLE table_name
ADD COLUMN column_name data_type [constraint];
```

### Example:

```sql
ALTER TABLE personDB
ADD COLUMN email VARCHAR(100) UNIQUE;
```

## 2️⃣ Drop a Column

```sql
ALTER TABLE table_name
DROP COLUMN column_name [CASCADE | RESTRICT];
```

### Example:

```sql
ALTER TABLE personDB
DROP COLUMN city;
```

- **CASCADE** → drops dependent objects
- **RESTRICT** → prevents drop if dependencies exist (default)

## 3️⃣ Modify Column Data Type

```sql
ALTER TABLE table_name
ALTER COLUMN column_name TYPE new_data_type;
```

### Example:

```sql
ALTER TABLE personDB
ALTER COLUMN name TYPE VARCHAR(100);
```

## 4️⃣ Rename a Column

```sql
ALTER TABLE table_name
RENAME COLUMN old_name TO new_name;
```

### Example:

```sql
ALTER TABLE personDB
RENAME COLUMN dob TO date_of_birth;
```

## 5️⃣ Rename the Table

```sql
ALTER TABLE old_table_name
RENAME TO new_table_name;
```

### Example:

```sql
ALTER TABLE personDB
RENAME TO employeeDB;
```

## 6️⃣ Add a Constraint

```sql
ALTER TABLE table_name
ADD CONSTRAINT constraint_name constraint_definition;
```

### Example:

```sql
ALTER TABLE personDB
ADD CONSTRAINT chk_salary CHECK (salary > 0);
```

## 7️⃣ Drop a Constraint

```sql
ALTER TABLE table_name
DROP CONSTRAINT constraint_name [CASCADE | RESTRICT];
```

### Example:

```sql
ALTER TABLE personDB
DROP CONSTRAINT chk_salary;
```
