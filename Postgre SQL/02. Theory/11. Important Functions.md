# PostgreSQL Commonly Used Functions

This document covers the most commonly used PostgreSQL functions categorized for easier reference.  
Each category has its own table with examples.

---

## 1. String Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `LENGTH(str)` | Returns length of a string | `SELECT LENGTH('hello'); -- 5` |
| 2 | `UPPER(str)` | Converts to uppercase | `SELECT UPPER('postgres'); -- POSTGRES` |
| 3 | `LOWER(str)` | Converts to lowercase | `SELECT LOWER('POSTGRES'); -- postgres` |
| 4 | `INITCAP(str)` | Capitalizes first letter of each word | `SELECT INITCAP('hello world'); -- Hello World` |
| 5 | `CONCAT(str1, str2, …)` | Concatenates multiple strings | `SELECT CONCAT('Post', 'greSQL'); -- PostgreSQL` |
| 6 | `SUBSTRING(str, from, len)` | Extracts substring | `SELECT SUBSTRING('Postgres', 1, 4); -- Post` |
| 7 | `TRIM(str)` | Removes leading/trailing spaces | `SELECT TRIM('  hello  '); -- hello` |
| 8 | `REPLACE(str, from, to)` | Replaces substring | `SELECT REPLACE('2025-09-06','-','/'); -- 2025/09/06` |
| 9 | `POSITION(substr in str)` | Finds index of substring | `SELECT POSITION('gres' IN 'Postgres'); -- 5` |

## 2. Numeric / Mathematical Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `ABS(x)` | Absolute value | `SELECT ABS(-10); -- 10` |
| 2 | `CEIL(x)` or `CEILING(x)` | Smallest integer ≥ x | `SELECT CEIL(4.2); -- 5` |
| 3 | `FLOOR(x)` | Largest integer ≤ x | `SELECT FLOOR(4.9); -- 4` |
| 4 | `ROUND(x, n)` | Rounds number to n decimals | `SELECT ROUND(15.678, 2); -- 15.68` |
| 5 | `POWER(x, y)` | x raised to power y | `SELECT POWER(2, 3); -- 8` |
| 6 | `SQRT(x)` | Square root | `SELECT SQRT(16); -- 4` |
| 7 | `MOD(x, y)` | Remainder of division | `SELECT MOD(10, 3); -- 1` |
| 8 | `RANDOM()` | Returns random value (0–1) | `SELECT RANDOM(); -- 0.534...` |

---

## 3. Date/Time Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `NOW()` | Current timestamp | `SELECT NOW();` |
| 2 | `CURRENT_DATE` | Current date | `SELECT CURRENT_DATE;` |
| 3 | `CURRENT_TIME` | Current time | `SELECT CURRENT_TIME;` |
| 4 | `AGE(timestamp1, timestamp2)` | Difference between dates | `SELECT AGE('2025-09-06', '2020-09-06'); -- 5 years` |
| 5 | `EXTRACT(field FROM date)` | Extracts year, month, etc. | `SELECT EXTRACT(YEAR FROM NOW());` |
| 6 | `DATE_TRUNC('unit', timestamp)` | Truncates date to unit | `SELECT DATE_TRUNC('month', NOW());` |
| 7 | `TO_CHAR(date, format)` | Formats date/time | `SELECT TO_CHAR(NOW(), 'YYYY-MM-DD');` |
| 8 | `MAKE_DATE(y, m, d)` | Constructs a date | `SELECT MAKE_DATE(2025, 9, 6);` |

---

## 4. Aggregate Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `COUNT(*)` | Counts rows | `SELECT COUNT(*) FROM employees;` |
| 2 | `SUM(col)` | Sum of column values | `SELECT SUM(salary) FROM employees;` |
| 3 | `AVG(col)` | Average value | `SELECT AVG(salary) FROM employees;` |
| 4 | `MIN(col)` | Minimum value | `SELECT MIN(salary) FROM employees;` |
| 5 | `MAX(col)` | Maximum value | `SELECT MAX(salary) FROM employees;` |
| 6 | `STRING_AGG(str, sep)` | Concatenates values with separator | `SELECT STRING_AGG(name, ', ') FROM employees;` |

---

## 5. Window (Analytic) Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `ROW_NUMBER()` | Assigns row number | `SELECT name, ROW_NUMBER() OVER(ORDER BY salary) FROM employees;` |
| 2 | `RANK()` | Assigns rank (with gaps) | `SELECT name, RANK() OVER(ORDER BY salary DESC) FROM employees;` |
| 3 | `DENSE_RANK()` | Rank without gaps | `SELECT name, DENSE_RANK() OVER(ORDER BY salary DESC) FROM employees;` |
| 4 | `NTILE(n)` | Divides rows into n groups | `SELECT name, NTILE(4) OVER(ORDER BY salary) FROM employees;` |
| 5 | `LAG(col)` | Value of previous row | `SELECT name, LAG(salary) OVER(ORDER BY id) FROM employees;` |
| 6 | `LEAD(col)` | Value of next row | `SELECT name, LEAD(salary) OVER(ORDER BY id) FROM employees;` |

---

## 6. Array Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `ARRAY[...]` | Create array | `SELECT ARRAY[1,2,3];` |
| 2 | `ARRAY_LENGTH(arr, dim)` | Length of array | `SELECT ARRAY_LENGTH(ARRAY[1,2,3], 1); -- 3` |
| 3 | `UNNEST(arr)` | Expands array to rows | `SELECT UNNEST(ARRAY['a','b']);` |
| 4 | `ARRAY_APPEND(arr, val)` | Adds element | `SELECT ARRAY_APPEND(ARRAY[1,2], 3); -- {1,2,3}` |
| 5 | `ARRAY_PREPEND(val, arr)` | Adds element at start | `SELECT ARRAY_PREPEND(0, ARRAY[1,2]); -- {0,1,2}` |
| 6 | `ARRAY_REMOVE(arr, val)` | Removes element | `SELECT ARRAY_REMOVE(ARRAY[1,2,2], 2); -- {1}` |

---

## 7. JSON Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `->` | Get JSON object field | `SELECT '{"name":"Rahul"}'::json->'name'; -- "Rahul"` |
| 2 | `->>` | Get JSON text field | `SELECT '{"name":"Rahul"}'::json->>'name'; -- Rahul` |
| 3 | `JSONB_BUILD_OBJECT(k, v, …)` | Build JSON object | `SELECT JSONB_BUILD_OBJECT('name','Rahul');` |
| 4 | `JSONB_ARRAY_ELEMENTS(jsonb)` | Expands array to rows | `SELECT JSONB_ARRAY_ELEMENTS('[1,2,3]'::jsonb);` |
| 5 | `JSONB_EXTRACT_PATH(jsonb, path…)` | Extracts nested element | `SELECT JSONB_EXTRACT_PATH('{"a":{"b":1}}'::jsonb,'a','b');` |

---

## 8. Conversion & Formatting Functions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `CAST(val AS type)` | Converts type | `SELECT CAST('10' AS INTEGER);` |
| 2 | `::` (shorthand cast) | Alternate type conversion | `SELECT '10'::INTEGER;` |
| 3 | `TO_CHAR(val, format)` | Converts to formatted string | `SELECT TO_CHAR(12345, '999,999'); -- 12,345` |
| 4 | `TO_DATE(str, format)` | Converts string to date | `SELECT TO_DATE('2025-09-06','YYYY-MM-DD');` |
| 5 | `TO_NUMBER(str, format)` | Converts string to number | `SELECT TO_NUMBER('12,345','99,999');` |

---

## 9. Conditional Expressions

| S. No. | Function | Usage / Meaning | Example |
|--------|----------|-----------------|---------|
| 1 | `CASE WHEN ... THEN ... ELSE ... END` | Conditional logic | `SELECT CASE WHEN salary > 50000 THEN 'High' ELSE 'Low' END FROM employees;` |
| 2 | `COALESCE(val1, val2, …)` | First non-null value | `SELECT COALESCE(NULL, NULL, 'Hello'); -- Hello` |
| 3 | `NULLIF(val1, val2)` | NULL if equal else val1 | `SELECT NULLIF(10, 10); -- NULL` |
| 4 | `GREATEST(val1, val2, …)` | Largest value | `SELECT GREATEST(1,5,3); -- 5` |
| 5 | `LEAST(val1, val2, …)` | Smallest value | `SELECT LEAST(1,5,3); -- 1` |
