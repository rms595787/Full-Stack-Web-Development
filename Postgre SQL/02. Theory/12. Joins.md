# Joins

| S.No. | Join Type  | Usage/Meaning                                 | Example                                 |
| ----- | ---------- | --------------------------------------------- | --------------------------------------- |
| 1     | INNER JOIN | Rows with matching values in both tables      | `FROM A INNER JOIN B ON A.id = B.id;`   |
| 2     | LEFT JOIN  | All rows from left table + matches from right | `FROM A LEFT JOIN B ON A.id = B.id;`    |
| 3     | RIGHT JOIN | All rows from right table + matches from left | `FROM A RIGHT JOIN B ON A.id = B.id;`   |
| 4     | FULL JOIN  | All rows if match exists in either table      | `FROM A FULL JOIN B ON A.id = B.id;`    |
| 5     | CROSS JOIN | Cartesian product of both tables              | `FROM A CROSS JOIN B;`                  |
| 6     | SELF JOIN  | Table joined with itself                      | `FROM A a JOIN A b ON a.id = b.ref_id;` |

## 📘 Types of Joins in SQL/PostgreSQL

### 1. INNER JOIN

- Returns only the rows where there is a match in both tables.
- Commonly used when you want intersection.

```sql
SELECT e.emp_id, e.name, d.dept_name
FROM employee e
INNER JOIN department d
ON e.dept_id = d.dept_id;
```

📌 Only employees with a department are shown.

### 2. LEFT JOIN (LEFT OUTER JOIN)

- Returns all rows from the left table and the matched rows from the right table.
- If no match → NULL is returned for right table columns.

```sql
SELECT e.emp_id, e.name, d.dept_name
FROM employee e
LEFT JOIN department d
ON e.dept_id = d.dept_id;
```

📌 Shows all employees, even those without a department (NULL in dept_name).

### 3. RIGHT JOIN (RIGHT OUTER JOIN)

- Returns all rows from the right table and the matched rows from the left table.
- If no match → NULL for left table columns.

```sql
SELECT e.emp_id, e.name, d.dept_name
FROM employee e
RIGHT JOIN department d
ON e.dept_id = d.dept_id;
```

📌 Shows all departments, even those without employees.

### 4. FULL JOIN (FULL OUTER JOIN)

- Returns all rows when there is a match in one of the tables.
- Combines results of LEFT JOIN + RIGHT JOIN.

```sql
SELECT e.emp_id, e.name, d.dept_name
FROM employee e
FULL JOIN department d
ON e.dept_id = d.dept_id;
```

📌 Shows all employees and all departments, matching where possible.

### 5. CROSS JOIN

- Returns the Cartesian product of both tables (all combinations).
- No ON condition is used.

```sql
SELECT e.name, d.dept_name
FROM employee e
CROSS JOIN department d;
```

📌 If 5 employees × 3 departments = 15 rows returned.

### 6. SELF JOIN

- Joins a table with itself (using aliases).
- Useful for hierarchical data (e.g., employees and managers).

```sql
SELECT e.name AS employee, m.name AS manager
FROM employee e
INNER JOIN employee m
ON e.manager_id = m.emp_id;
```

📌 Shows employees with their managers.
