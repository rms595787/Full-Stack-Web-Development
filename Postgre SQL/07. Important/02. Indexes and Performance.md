# PostgreSQL – Indexes & Performance

## 📑 Types of Indexes
| Index Type | Explanation | Example |
|------------|-------------|---------|
| **B-Tree** (default) | Good for equality & range lookups | `CREATE INDEX idx_name ON employees(name);` |
| **Hash** | Fast equality search, no range queries | `CREATE INDEX idx_hash ON employees USING HASH(email);` |
| **GIN (Generalized Inverted Index)** | Full-text search, JSONB | `CREATE INDEX idx_json ON docs USING GIN(data);` |
| **GiST (Generalized Search Tree)** | Geospatial, full-text, custom ops | `CREATE INDEX idx_geom ON locations USING GIST(geom);` |
| **BRIN (Block Range Index)** | Large sequential data (time-series) | `CREATE INDEX idx_date ON logs USING BRIN(created_at);` |
| **Unique Index** | Enforces uniqueness | `CREATE UNIQUE INDEX idx_emp_email ON employees(email);` |

## 🚀 Performance Tips
| Command/Concept | Explanation | Example |
|-----------------|-------------|---------|
| `EXPLAIN query;` | Shows query plan | `EXPLAIN SELECT * FROM employees WHERE id=10;` |
| `EXPLAIN ANALYZE query;` | Runs query & shows execution details | `EXPLAIN ANALYZE SELECT * FROM employees WHERE id=10;` |
| `ANALYZE;` | Updates planner statistics | `ANALYZE employees;` |
| `VACUUM;` | Reclaims space, updates stats | `VACUUM employees;` |
| `VACUUM FULL;` | Aggressive cleanup (locks table) | `VACUUM FULL employees;` |
