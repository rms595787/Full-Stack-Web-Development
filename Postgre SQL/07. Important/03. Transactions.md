# PostgreSQL – Transactions

## ⚡ Basics

| Command                       | Explanation                       | Example                                                                |
| ----------------------------- | --------------------------------- | ---------------------------------------------------------------------- |
| `BEGIN; ... COMMIT;`          | Start & commit transaction        | `BEGIN; UPDATE accounts SET balance = balance-100 WHERE id=1; COMMIT;` |
| `ROLLBACK;`                   | Undo all operations since `BEGIN` | `ROLLBACK;`                                                            |
| `SAVEPOINT name;`             | Create checkpoint                 | `SAVEPOINT sp1;`                                                       |
| `ROLLBACK TO SAVEPOINT name;` | Rollback to checkpoint            | `ROLLBACK TO SAVEPOINT sp1;`                                           |
| `RELEASE SAVEPOINT name;`     | Remove savepoint                  | `RELEASE SAVEPOINT sp1;`                                               |

---

## 🔒 Isolation Levels

| Level                        | Explanation                                                   |
| ---------------------------- | ------------------------------------------------------------- |
| **READ UNCOMMITTED**         | Reads uncommitted changes (Postgres treats as READ COMMITTED) |
| **READ COMMITTED** (default) | Sees only committed rows                                      |
| **REPEATABLE READ**          | Same snapshot for whole transaction                           |
| **SERIALIZABLE**             | Strictest, ensures no anomalies                               |

```sql
-- Example: Set isolation level
BEGIN TRANSACTION ISOLATION LEVEL SERIALIZABLE;
```
