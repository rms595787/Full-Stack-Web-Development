# To install React.js

- use command -> npm create vite@latest
- this command will create a react project
- then change directory and go to the project
- then open vs code terminal and run this command -> npm i
- this command will install packages on which react.js depends on

# Let's Understand which package is used for what

**node_module package**

- inside our react project first package is "node_modules" here we have all packages and modules on which react.js depends on
- and later if we decide to install third party packages to work with they will go to this "node_module" package

**public folder**

- in here we will put all our asset files like html, css, etc.

**src folder**

- we will be working here 90% of our time on react project
- **assets folder**
- - inside src folder we have assets folder where we put all our images, icons, etc.

- **App.css and index.css files**
- - in these files we are going to do our styling of the project

- **App.jsx and main.jsx files**
- - these are components files

- **.gitignore file**
- - this is a github file used for ignoring certain files and folders from uploading on github
- - for ex - node_module is a very large file if we do not want to upload on github we can just ignore it

- **eslint.config.js file**
- - this file is used for configuring eslint

- **index.html file**
- - it is the heart of our project here we have our html content and connections of all files which we will create in this project

- **package.json file**
- - this contains meta data about our project like project name, version, description and scripts
- - it list the dependencies and version on which our project needs to run

- **package-lock.json file**
- - automatically generated and updated when install or modified dependencies
- - it locks the version of all of the packages and their dependencies to ensure the consistent and reproduce while built environment

- **Readme.md file**
- - it holds the information about what our project is all about

- **vite.config.js**
- - it is the configuration file for vite itself
