# io.on('connection',callback)

io.on is a method used to register event listeners for different events that occur on the server-side. The io object represents the main Socket.IO server instance.

## Let's understand this

When we call `io.on('connection',callback)` we're telling Socket.IO to listen for a connection event, which occurs whenever a new client establishes a connection with the server. The callback function will be called/invoked whenever this event occurs, and it will receive a socket object representing the connetion to the client.

Similarly, we can use `socket.on` on the socket object to listen for events specific to that individual client connection.

## This is how we connect to socket.io using `backend`

```js
// connection event handler
io.on("connection", (socket) => {
  console.log("User Connected To (server)");
  console.log(socket);
});
```

### Things we need to do, to make this event working

1. Create this event listener in `index.js` file

```js
io.on("connection", (socket) => {
  console.log("User Connected on (server) ✅");
  console.log(socket);
});
```

2. You have to connect this `scipt` in your `index.html` file

- this "/socket.io/socket.io.js" contains whole socket.io library
- and it is necessary to load this library in your frontend or client server -> meaning in your html file

- You’re telling the browser to load the Socket.IO client-side JavaScript file served automatically by your Node.js + Express + Socket.IO server.

```html
<script src="/socket.io/socket.io.js"></script> - here we imported socket.io client side javaScript file

- now we have to create instance of this imported library
<scirpt type="module">
    const socket = io();
</script>

```

- now your events are connected to your frontend and everyting works properly

## Disconnecting specific client not the whole `socket.io server` from the `backend`

```js
io.on("connection", (socket) => {
  console.log("User Connected To (server) ✅");

  // we are not disconnecting "whole socket.io server" instead we are disconneting the "client of this socket.io server"
  socket.on(
    "disconnect",
    () => console.log("User Disconnected From (server) ❌")
    // when you close the page you can see that client is been disconnected while our server is still running
  );
});
```

## Disconnecting specific client not the whole `socket.io server` from the `frontend`

```js
<script type="module">
    const socket = io();    // here we instantiated
    // now we will make connection with socket.io from frontend or client side
    socket.on("connect", () => {
        console.log("Connected to (client) ✅");    // this will be visible on console of browser
        
        // here we are calling the disconnect
        // Disconnect from server after 5 second
        setTimeout(() => {
            socket.disconnect();
        }, 5000);
    })
    
    // Trigger when disconnected
    socket.on("disconnect", () => {
        console.log("Disconnected from (client) ❌");
    })
</script>
```
