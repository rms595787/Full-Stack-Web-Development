# 📌 What is Projection in MongoDB? - [Link](https://www.mongodb.com/docs/manual/reference/mql/projection/)

Projection in MongoDB is the process of selecting only specific fields from a document when you query a collection.

- By default, MongoDB returns all fields of a document.
- With projection, you can include or exclude fields from the output.

## 📌 Syntax

```js
db.collection.find(query, projection);
```

- query → filter condition (like WHERE in SQL).
- projection → specify which fields to include/exclude.

## 📌 Example

Suppose we have a collection students:

```js
{
  "_id": 1,
  "name": "Rahul",
  "age": 22,
  "course": "Computer Science",
  "marks": 89
}
```

✅ Return only name and course

```js
db.students.find({}, { name: 1, course: 1 });
```

## Result:

```js
{ "_id": 1, "name": "Rahul", "course": "Computer Science" }
```

👉 By default, \_id is always included. To exclude it:

```js
db.students.find({}, { name: 1, course: 1, _id: 0 });
```

## 📌 Projection Rules

- Inclusion (1) → db.collection.find({}, { field: 1 })
  - Returns only the specified fields.
- Exclusion (0) → db.collection.find({}, { field: 0 })
  - Excludes the specified fields and returns all others.
- Cannot mix inclusion & exclusion together
  - ❌ db.students.find({}, { name: 1, age: 0 }) → Not allowed.
  - ✅ Exception: \_id can be excluded while including fields.

## 📌 Advanced Projections

MongoDB also supports:

- Dot notation → project nested fields

```js
db.students.find({}, { "address.city": 1 });
```

- Array projection with $elemMatch → return only matching array element

```js
db.students.find(
  { marks: { $gt: 80 } },
  { name: 1, marks: { $elemMatch: { $gt: 80 } } }
);
```

- Aggregation projection ($project) in pipelines

```js
db.students.aggregate([{ $project: { name: 1, total: { $sum: "$marks" } } }]);
```

✅ In short: Projection = selecting only the fields you care about, instead of fetching everything.
