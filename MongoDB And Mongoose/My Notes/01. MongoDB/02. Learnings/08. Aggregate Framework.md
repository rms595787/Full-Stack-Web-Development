# Aggregate Framework

Aggregation is a powerful framework for complex operations

like filtering, grouping, sorting, reshaping, and summarizing data in a flexible way via pipeline.

<img src="./_images/Aggregate_Framework.png" width="400">

Syntax

```js
db.collection.aggregate([
    {stage1},
    {stage2}...
],{option})
```

Example

```js
db.orders.aggregate([
  // stage 1: Filter pizza order documents by pizza size
  {
    $match: { size: "medium" },
  },

  // stage 2: Group remaining documents by pizza name and calculate total quantity
  {
    $group: { _id: "$name", totalQuantity: { $sum: "quantity" } },
  },
]);
```

# Most commonly used stages in MongoDB aggregation:

- $match → Filters documents (like find)
- $group → Groups documents and applies accumulators
- $project → Controls which fields to show
- $sort → Sorts documents
- $limit → Restricts number of results
- $sortByCount → Groups + counts + sorts in one step
- $unwind → Deconstructs arrays into multiple docs
- $lookup → Performs join between collections
- $addFields → Adds or modifies fields
- $count → Counts number of documents
- $skip → Skips certain number of documents
- $out → It writes the result of the aggregation pipeline to a new (or existing) collection.

## 1. Group stage

Purpose → Group documents by a field and perform aggregate calculations.

Sytanx

```js
db.collection.aggregate([
  {
    $group: {
      _id: "$category", // Group documents by the "category" field.
      // The `$` symbol means "use the value of this field".

      totalAmount: { $sum: "$amount" }, // Sum of "amount" for each category.
      averageAmount: { $avg: "$amount" }, // Average of "amount" for each category.
      minAmount: { $min: "$amount" }, // Minimum "amount" value in the group.
      maxAmount: { $max: "$amount" }, // Maximum "amount" value in the group.

      amountList: { $push: "$amount" }, // Creates an array of ALL "amount" values in the group.
      uniqueAmounts: { $addToSet: "$amount" }, // Creates an array of UNIQUE "amount" values only.
    },
  },
]);
```

### 🔑 Key Notes:

- \_id inside $group → defines the grouping key. Here, all docs with the same category value will be grouped together.
- Accumulator operators like $sum, $avg, $min, $max, $push, $addToSet → define how values inside each group are aggregated.
- $push vs $addToSet:

  - $push → keeps duplicates, preserves all values.
  - $addToSet → removes duplicates (acts like a set).

### Use Case: Find no. of cars of each brand (we have to group)


## Count Stage

Purpose → Returns a count of documents that match a filter.

### Use Case: Print Total no. of Hyundai Cars

```js
db.cars.aggregate([
  { $match: { maker: "Hyundai" } }, // stage 1 which is match - will find all cars with maker hyundai
  { $count: "Total_cars" }, // stage 2 this will get operated on the result match gives - will count total_cars
]);
```

### Use Case2: Count no. of diesel and petrol cars of Hyundai brand

```js
// This groups all the hyundai cars according to there fuel_type
db.cars.aggregate([
  { $match: { maker: "Hyundai" } },
  { $group: { _id: "$fuel_type", Total_cars: { $sum: 1 } } },
]);

// This groups all hyundai cars but only petrol and diesel fuel_type
db.cars.aggregate([
  { $match: { maker: "Hyundai", fuel_type: { $in: ["Petrol", "Diesel"] } } },
  { $group: { _id: "$fuel_type", Total_cars: { $sum: 1 } } },
]);
```

## $addFields Stage

#### ✅ Purpose

- Adds new fields or modifies existing ones in the documents passing through the pipeline.
- Similar to $project, but instead of reshaping the document, it adds fields while keeping the rest unchanged.

Syntax

```js
{
  $addFields: {
    newField: <expression>,
    existingField: <expression>
  }
}
```

Example

```js
db.cars.aggregate([
  {
    $addFields: {
      fullName: { $concat: ["$maker", " ", "$model"] }, // combine fields
      isElectric: { $eq: ["$fuel_type", "Electric"] }, // boolean flag
    },
  },
]);
```

`These fullName and isElectric field got added in every document`

<img src="./_images/Aggregate_addFields.png" width=700>

### Use Case: Print details of cars with price in lakhs (15 lakhs)

```js
db.cars.aggregate([
  { $project: { model: 1, _id: 0, price: 1 } },
  {
    $addFields: {
      price_in_lakhs: {
        // $toString converts the double into string, since we can only concat string we must convert double to string in order to get this result
        $concat: [{ $toString: { $divide: ["$price", 100000] } }, " lakhs"],
      },
    },
  },
]);
```

## $lookup Stage

#### ✅ Purpose

- Performs a left outer join between two collections.
- Lets you merge related data from another collection into your pipeline.

Syntax

```js
{
  $lookup: {
    from: "<foreign_collection>",
    localField: "<field_in_current_collection>",
    foreignField: "<field_in_foreign_collection>",
    as: "<output_array_field>"
  }
}
```

Let's create a new collection `dealers` to understand it better

`Collection dealer`

```js
db.dealers.insertMany([
  { dealer_id: 1, location: "Mumbai", dealer_name: "Hyundai Mumbai Central" },
  { dealer_id: 2, location: "Delhi", dealer_name: "Maruti Delhi South" },
  { dealer_id: 3, location: "Bangalore", dealer_name: "Tata Bangalore West" },
]);
```

`Now join cars with dealers based on owners’ location 👇`

```js
db.cars.aggregate([
  {
    $lookup: {
      from: "dealers", // foreign collection
      localField: "owners.location", // owners' location from cars
      foreignField: "location", // location in dealers
      as: "dealerInfo", // new field
    },
  },
  {
    $project: {
      maker: 1,
      model: 1,
      "owners.name": 1,
      "owners.location": 1,
      dealerInfo: 1,
    },
  },
]);
```

`✅ Example Output (for Hyundai Creta with owner in Mumbai):`

```js
{
  maker: "Hyundai",
  model: "Creta",
  owners: [
    { name: "Raju", location: "Mumbai" },
    { name: "Shyam", location: "Delhi" }
  ],
  dealerInfo: [
    { dealer_id: 1, location: "Mumbai", dealer_name: "Hyundai Mumbai Central" },
    { dealer_id: 2, location: "Delhi", dealer_name: "Maruti Delhi South" }
  ]
}
```

This example output got created for every car

#### 📝 Notes

- Performs left outer join between collections.
- Merges related documents into an array.
- Can also be extended with pipeline syntax for filtering and projecting join results.

## $out Stage

#### 🔹 What $out Does

- It writes the result of the aggregation pipeline to a new (or existing) collection.
- Effectively, it takes the final output of the pipeline and stores it permanently instead of just returning it.

Syntax

```js
db.collection.aggregate([
  { <stages...> },
  { $out: "targetCollectionName" }
])
```

Example

```js
db.cars.aggregate([{ $match: { maker: "Hyundai" } }, { $out: "hyundaiCars" }]);
```

Now if you run

```js
db.hyundaiCars.find();
```

👉 You’ll see only Hyundai cars stored in that new collection.

#### 🔑 Notes

- $out overwrites the target collection if it already exists.
- If you want to merge instead of overwrite, use $merge (introduced in MongoDB 4.2).
- $out must be the last stage in the pipeline.

<img src="./_images/Aggregate_out_stage.png">

### Use Case: After aggregating, store the result in an another collection 'hyundai_cars'

```js
db.cars.aggregate([
  { $match: { maker: "Hyundai" } },
  { $project: { _id: 0, CarName: { $concat: ["$maker", " ", "$model"] } } },
  { $out: "hyundai_cars" },
]);
```
