# Indexes

An INDEX is a data structure that improves the speed of query operations by allowing the database to quickly locate and access the required data without scanning every document in a collection.  
Stores the indexed fields in a sorted order, along with pointers to the actual documents in the collection.

- A data structure (B-Tree) that improves query performance.
- Without indexes → MongoDB does a COLLSCAN (scans all documents).

Syntax

```js
db.collection.createIndex(
    {
        <field1>:<type1>,
        <field2>:<type2>,...
    },{<options>}
)
```

Some Examples

```js
db.cars.createIndex({ maker: 1 });

db.cars.createIndex({ mode: 1 }, { unique: true });

db.cars.dropIndex("maker");

db.cars.getIndexes();
```

# Types of Indexes

1. Default \_id Index

   - Every collection has an automatic unique index on \_id.

2. Single Field Index

   - Index on one field.
   - Example:
     ```js
     db.users.createIndex({ name: 1 }); // Ascending
     db.users.createIndex({ age: -1 }); // Descending
     ```

3. Compound Index

   - Index on multiple fields (order matters).
   - Example:
     ```js
     db.users.createIndex({ name: 1, age: -1 });
     ```

4. Multikey Index

   - For array fields. Creates index entries for each array element.
   - Example:

     ```js
     db.users.createIndex({ tags: 1 });
     ```

5. Text Index

   - Supports text search ($text).
   - Example:
     ```js
     db.articles.createIndex({ content: "text", title: "text" });
     ```

6. Hashed Index

   - Stores hashed values → used for sharding & equality queries.
   - Example:
     ```js
     db.users.createIndex({ email: "hashed" });
     ```

7. Wildcard Index ($\*\*)

   - Index all fields or dynamic fields.
   - Example:
     ```js
     db.data.createIndex({ "$**": 1 });
     ```

8. Unique Index

   - Prevents duplicate values.
   - Example:
     ```js
     db.users.createIndex({ email: 1 }, { unique: true });
     ```

9. Partial Index

   - Index only documents that match a filter.
   - Example:
     ```js
     db.users.createIndex(
       { status: 1 },
       { partialFilterExpression: { status: { $eq: "active" } } }
     );
     ```

10. TTL (Time-to-Live) Index
    - Auto-deletes documents after a given time.
    - Example:
      ```js
      db.sessions.createIndex({ createdAt: 1 }, { expireAfterSeconds: 3600 });
      ```

## ⚡ Important Index Concepts

- Covered Query → If all fields in query & projection are in the index → no need to access documents.
- Index Cardinality → High cardinality fields (many unique values) benefit more.
- Index Order → Matters in compound indexes ({ a:1, b:1 } ≠ { b:1, a:1 }).
- Index Size → Indexes take memory (RAM). Too many indexes slow down writes.
- Sparse vs Dense Indexes → Sparse indexes skip documents without the field.
- Hint → You can force MongoDB to use a specific index:

  ```js
  db.users.find({ name: "Amit" }).hint({ name: 1 });
  ```

## Performance Considerations:
- `Impace on write Operations`: While indexes speed up reads, they can slow down insertions, updates, and deletioncs because the indexes need to be maintained.
- `Indexing Large Collections`: Learn about the considerations when indexing large collections, such as index size and the impact on RAM
