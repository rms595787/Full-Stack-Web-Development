MongoDB has several ways to **count documents**, but they differ in accuracy, performance, and use cases. Here’s a breakdown of the main ones:

---

### 1. **`count()` (deprecated)**

```js
db.collection.count(query, options);
```

- **Use case**: Quickly count documents matching a query.
- **Limitations**: Deprecated because it can return inaccurate results in sharded clusters or with certain query filters.
- **Example**:

```js
db.users.count({ age: { $gt: 18 } });
```

---

### 2. **`countDocuments()`**

```js
db.collection.countDocuments(query, options);
```

- **Use case**: Accurate count of documents that match a filter.
- **Behavior**: Executes a full query and counts documents, so it’s more reliable but slower for very large datasets.
- **Example**:

```js
db.users.countDocuments({ age: { $gt: 18 } });
```

---

### 3. **`estimatedDocumentCount()`**

```js
db.collection.estimatedDocumentCount(options);
```

- **Use case**: Fast estimate of total documents in a collection (ignores filters).
- **Behavior**: Uses metadata (collection stats) instead of scanning documents.
- **Best for**: Getting a quick total count of all documents.
- **Example**:

```js
db.users.estimatedDocumentCount();
```

---

### 4. **Aggregation with `$count`**

```js
db.collection.aggregate([
  { $match: { age: { $gt: 18 } } },
  { $count: "total" },
]);
```

- **Use case**: When you need counting as part of a pipeline (e.g., filtering, grouping, etc.).
- **Flexible**: Can combine with other stages like `$group`, `$sort`, `$lookup`.
- **Example**:

```js
db.orders.aggregate([
  { $match: { status: "delivered" } },
  { $count: "deliveredOrders" },
]);
```

---

### ⚡ Quick Comparison

| Function                   | Supports Query Filter? | Accuracy      | Performance | Notes                      |
| -------------------------- | ---------------------- | ------------- | ----------- | -------------------------- |
| `count()` (deprecated)     | ✅ Yes                 | ⚠️ Not always | ⚡ Fast     | Avoid in new code          |
| `countDocuments()`         | ✅ Yes                 | ✅ Accurate   | 🐢 Slower   | Use for filtered counts    |
| `estimatedDocumentCount()` | ❌ No (all docs only)  | ⚠️ Estimate   | ⚡ Fast     | Good for large collections |
| `$count` (aggregation)     | ✅ Yes (via `$match`)  | ✅ Accurate   | 🐢 Depends  | Best for complex pipelines |

---

👉 Rule of thumb:

- Use **`countDocuments()`** when you need accuracy with filters.
- Use **`estimatedDocumentCount()`** for a fast, rough total.
- Use **`$count`** if you’re already in an aggregation pipeline.
