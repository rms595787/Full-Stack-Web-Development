# MongoDB – Indexes & Performance

## 📑 Types of Indexes

| Index Type         | Explanation                      | Example                                                                                  |
| ------------------ | -------------------------------- | ---------------------------------------------------------------------------------------- |
| Single Field       | Speeds up queries on one field   | `db.users.createIndex({ email: 1 })`                                                     |
| Compound           | Index on multiple fields         | `db.orders.createIndex({ customerId: 1, date: -1 })`                                     |
| Unique             | Ensures uniqueness               | `db.users.createIndex({ email: 1 }, { unique: true })`                                   |
| Sparse             | Indexes only docs with the field | `db.users.createIndex({ nickname: 1 }, { sparse: true })`                                |
| Partial            | Index subset of docs             | `db.users.createIndex({ status: 1 }, { partialFilterExpression: { status: "active" } })` |
| Text               | Full-text search                 | `db.articles.createIndex({ content: "text" })`                                           |
| TTL (Time-to-Live) | Auto-delete docs after time      | `db.sessions.createIndex({ createdAt: 1 }, { expireAfterSeconds: 3600 })`                |

---

## 🚀 Performance Tools

| Command                                               | Explanation               |
| ----------------------------------------------------- | ------------------------- |
| `db.collection.explain("executionStats").find(query)` | See how query is executed |
| `db.collection.stats()`                               | Collection stats          |
| `db.currentOp()`                                      | Show current operations   |
| `db.serverStatus()`                                   | Server metrics            |

---

## 🛠 Example

```js
// Check query plan
db.users.find({ email: "test@example.com" }).explain("executionStats");

// Drop index
db.users.dropIndex("email_1");

// List indexes
db.users.getIndexes();
```

# ⚡ Best Practices

- Always index fields used in queries, sorting, and joins ($lookup).
- Avoid too many indexes (they slow down writes).
- Use compound indexes wisely (order matters).
- Use `covered queries` (query returns only indexed fields) for max speed.
