# MongoDB – Transactions - [Link](https://www.google.com/search?client=safari&rls=en&q=mongodb+transactions&ie=UTF-8&oe=UTF-8)

## ⚡ Basics

- Transactions group multiple operations into one atomic unit.
- Supported on **replica sets and sharded clusters**.
- ACID properties.

---

## 🔹 Usage in Node.js

```js
const session = client.startSession();

try {
  session.startTransaction();

  await db
    .collection("accounts")
    .updateOne({ _id: 1 }, { $inc: { balance: -100 } }, { session });
  await db
    .collection("accounts")
    .updateOne({ _id: 2 }, { $inc: { balance: 100 } }, { session });

  await session.commitTransaction();
} catch (e) {
  await session.abortTransaction();
} finally {
  await session.endSession();
}
```

## 🛠 Commands

| Command               | Description             |
| --------------------- | ----------------------- |
| `startTransaction()`  | Start a new transaction |
| `commitTransaction()` | Save all changes        |
| `abortTransaction()`  | Rollback changes        |

# ⚡ Best Practices

- Keep transactions short (locks affect performance).
- Avoid long-running transactions.
- Index fields used inside transactions.
