# MongoDB – User & Role Management

## 🔑 Create Users

```js
use admin
db.createUser({
  user: "appUser",
  pwd: "secret123",
  roles: [{ role: "readWrite", db: "mydb" }]
});
```

## 🔐 Built-in Roles

| Role           | Scope   | Description                        |
| -------------- | ------- | ---------------------------------- |
| `read`         | DB      | Read only                          |
| `readWrite`    | DB      | Read and write                     |
| `dbAdmin`      | DB      | Admin tasks on DB (indexes, stats) |
| `userAdmin`    | DB      | Manage users & roles               |
| `clusterAdmin` | Cluster | Full cluster control               |
| `root`         | Cluster | Superuser                          |

## 👥 Managing Users

| Command         | Example                                        |
| --------------- | ---------------------------------------------- |
| List users      | `db.getUsers()`                                |
| Drop user       | `db.dropUser("appUser")`                       |
| Update password | `db.updateUser("appUser", { pwd: "newPass" })` |

# ⚡ Best Practices

- Use least privilege principle.
- Always create a separate user for applications (not admin).
- Enable authentication in production (--auth).
