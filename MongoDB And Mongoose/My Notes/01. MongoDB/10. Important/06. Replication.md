# MongoDB – Replication - [Link](https://www.mongodb.com/docs/manual/replication/)

## 🔄 What is Replication?

- Replication = multiple copies of data across servers.
- Ensures **high availability** and **redundancy**.
- Consists of **replica set**:
  - Primary (accepts writes).
  - Secondaries (replicate data).
  - Arbiter (breaks ties in elections, no data).

---

## 🛠 Basic Setup

```bash
mongod --replSet "rs0" --port 27017 --dbpath /data/db1
mongod --replSet "rs0" --port 27018 --dbpath /data/db2
mongod --replSet "rs0" --port 27019 --dbpath /data/db3

mongosh
rs.initiate({
  _id: "rs0",
  members: [
    { _id: 0, host: "localhost:27017" },
    { _id: 1, host: "localhost:27018" },
    { _id: 2, host: "localhost:27019" }
  ]
});
```

## 📊 Commands

| Command                  | Description              |
| ------------------------ | ------------------------ |
| `rs.status()`            | Check replication status |
| `rs.add("host:port")`    | Add a member             |
| `rs.remove("host:port")` | Remove a member          |

# ⚡ Best Practices

- Always deploy an odd number of members (3, 5, 7).
- Place members in different availability zones.
- Use hidden nodes for backups/reporting.
