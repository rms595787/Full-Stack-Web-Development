# MongoDB – Connectivity

## 🌍 Connection String Formats

| Format             | Example                                                      | Use Case              |
| ------------------ | ------------------------------------------------------------ | --------------------- |
| Standard           | `mongodb://username:password@host:port/db`                   | Local, single node    |
| SRV (DNS seedlist) | `mongodb+srv://user:pass@cluster0.mongodb.net/mydb`          | MongoDB Atlas (cloud) |
| With options       | `mongodb://localhost:27017/mydb?retryWrites=true&w=majority` | Fine-tuning           |

---

## 🔌 Connecting with Drivers

### Node.js (`mongodb` driver)

```js
const { MongoClient } = require("mongodb");
const uri = "mongodb+srv://user:pass@cluster0.mongodb.net/mydb";
const client = new MongoClient(uri);

async function main() {
  await client.connect();
  const db = client.db("mydb");
  const users = db.collection("users");
  console.log(await users.findOne({}));
  await client.close();
}
main();
```

### Python (`pymongo`)

```python
from pymongo import MongoClient

client = MongoClient("mongodb://localhost:27017/")
db = client["mydb"]
users = db["users"]

print(users.find_one({}))
```

### CLI (`mongosh`)

```bash
mongosh "mongodb+srv://cluster0.mongodb.net/mydb" --username user
```

## ⚡ Best Practices

- Store connection string in environment variables (.env file).
- Always use connection pooling (handled automatically by modern drivers).
- Use w=majority in connection options for reliable writes.
